What is Canvas?

Canvas is a 2D drawing API. Essentially the browser gives you a rectangular area on the screen that you can draw into.
You can draw lines, shapes, images, text.
Canvas was originally created by Apple for its Dashboard widgets, but it has since been adopted by every major browser vendor and is now part of the HTML 5 spec.

Comparing to the rest of the web

Let's find out where it fits in with the rest of the web?

There are four ways to draw things on the web:

  • Canvas
  • SVG
  • CSS
  • DOM animation
There are four ways to draw things on the web: Canvas, SVG, CSS, and direct DOM animation.

SVG

SVG is a vector API that draws shapes. Each shape has an object that you can attach event handlers to. If you zoom in the shape stays smooth, whereas Canvas would become pixelated.

CSS

CSS is about styling DOM elements. Since there are no DOM objects for things you draw in Canvas you can't use CSS to style it. CSS will only affect the rectangular area of the Canvas itself, so you can set a border and background color, but that's it.

DOM animation

The DOM defines an object for everything on the screen. DOM animation, either by using CSS or JavaScript to move objects around, can be smoother in some cases than doing it with Canvas, but it depends on your browser implementation.

The Document Object Model, defines an object for everything on the screen. DOM animation, either by using CSS or JavaScript to move objects around, can be smoother in some cases than doing it with Canvas, but it depends on your browser implementation.

Usage

  • Use SVG when you have existing shapes that you want to render to the screen.
  • Use CSS or DOM animation when you have large static areas that you wish to animate, or if you want to use 3D transforms.
  • UseCanvas for charts, graphs, dynamic diagrams, and of course video games.

So when should you use Canvas over SVG, CSS or DOM elements? Well, Canvas is lower level than those others so you can have more control over the drawing and use less memory, but at the cost of having to write more code. Use SVG when you have existing shapes that you want to render to the screen, like a map that came out of Adobe Illustrator. Use CSS or DOM animation when you have large static areas that you wish to animate, or if you want to use 3D transforms. For charts, graphs, dynamic diagrams, and of course video games, Canvas is a great choice.

Paths

Let's talk about paths.

Paths are shapes created by a bunch of straight or curved line segments. In Canvas you must first define a path with beginPath(), then you can fill it, stroke it, or use it as a clip. You define each line segment with functions like moveTo(), lineTo(), and bezierCurveTo().

Canvas only directly supports the rectangle shape. To draw any other shape you must draw it yourself using a path. Paths are shapes created by a bunch of straight or curved line segments. In Canvas you must first define a path with beginPath(), then you can fill it, stroke it, or use it as a clip. You define each line segment with functions like moveTo(), lineTo(), and bezierCurveTo().

Code for the shape above:

c.fillStyle = 'red';
  c.beginPath();
  c.moveTo(10,30);
  c.bezierCurveTo(50,90,159,-30,200,30);
  c.lineTo(200,90);
  c.lineTo(10,90);
  c.closePath();
  c.fill();
  c.lineWidth = 4;
  c.strokeStyle = 'black';
  c.stroke();
This example draws a shape with a move to, followed by a bezier curve segment, then some lines. After creating the path it fills and strokes it.

Images

Canvas can also draw images

Canvas can draw images with the drawImage function. There are several forms of drawImage:
You can draw the image directly to the screen at normal scale, or stretch and slice it how you like.

We can draw images with the drawImage function. There are several forms of drawImage. You can draw the image directly to the screen at normal scale, or stretch and slice it how you like. Slicing and stretching images can be very handy for special effects in games, because image interpolation is often much faster than other kinds of scaling. 

  ctx.drawImage(img, 0,0); //normal drawing
  ctx.drawImage(img, //draw stretched
	  0,0,66,66, //source (x,y,w,h)
	  100,0,100,100//destination (x,y,w,h)
	  );
  ctx.drawImage(img, //draw a slice
	  20,10,20,20, //source coords (x,y,w,h)
	  250,0,250,50//destination coords (x,y,w,h)
	  );
To stretch an image you must specify the source and destination coordinates. The source coordinates tell drawImage where to pull the pixels from in the image. The destination coordinates tell the drawImage where to put the pixels on the screen. By changing the w and h coords you can stretch and shrink the image. Slicing is the same thing, but using source coordinates that don't cover the entire image. When you take a slice of an image be sure you don't go outside the source bounds or else the image will disappear. For example, if you drag the source width past 46, then it will try to access pixels beyond the right edge of the image. Using a negative source x coordinate will do the same

Text

It's possible to draw text with Canvas as well.

The font attribute is the same as its CSS equivalent, so you can set the style, size, and font family. The fillText(string,x,y) function draws using baseline of the text, not the top.

The font attribute is the same as its CSS equivalent, so you can set the style, size, and font family. Note that the fillText(string,x,y) function draws using baseline of the text, not the top. If you put your text at 0,0 then it will be drawn off the top of the screen. Be sure to lower the y by an appropriate amount 

  ctx.fillStyle = "black";
  ctx.font = "italic "+44+"pt Arial ";
  ctx.fillText("this is text", 11,123);

Gradient

And finally, let's see how gradients can be created using canvas. 

  var grad = ctx.createLinearGradient(0,0,200,0);
  grad.addColorStop(0, "white");
  grad.addColorStop(0.5, "red");
  grad.addColorStop(1, "black");
					
  ctx.fillStyle = grad;
  ctx.fillRect(0,0,400,200);
Here's a linear gradient. You see Canvas can also fill shapes with gradients instead of colors. An important thing to notice here is that gradient is painted in the coordinate system that the shape is drawn in, not the internal coordinates of the shape.
